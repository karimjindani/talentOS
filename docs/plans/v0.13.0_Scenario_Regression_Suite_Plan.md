# v0.13.0 Scenario-Based Regression Suite Enhancement Plan

## Summary

Enhance TalentOS regression testing from mostly unit/helper tests into an Ops-controlled scenario
regression suite. The Ops Console allows running the full regression suite or a specific logical area,
and displays clear pass/fail/skip counts after each run.

Use `v0.13.0` because this adds a new testing capability, Ops UI/API behavior, scripts and
documentation.

## Key Changes

- Add scenario test categories:
  - `unit`
  - `auth`
  - `applicant`
  - `admin`
  - `programs`
  - `tenant`
  - `dashboard`
  - `storage`
  - `ops`
  - `all`
- Add npm scripts for each category:
  - `regression:unit`
  - `regression:auth`
  - `regression:applicant`
  - `regression:admin`
  - `regression:programs`
  - `regression:tenant`
  - `regression:dashboard`
  - `regression:storage`
  - `regression:ops`
  - `regression:all`
- Extend the Ops job API to accept:

```json
{ "kind": "regression", "area": "all" }
```

- Add Ops UI controls:
  - Regression Area selector.
  - Run Selected Area.
  - Run Full Regression.
  - Summary counts for total, passed, failed, skipped and duration.
- Keep existing cleanup and reset behavior.
- Ensure scenario-generated data is tagged with `RegressionDataMarker`.

## Initial Automated Scenarios

1. Auth smoke:
   - Keycloak discovery.
   - Org Admin reaches admin portal.
   - Applicant reaches application page.
   - Accepted applicant reaches dashboard.
   - Org Admin reaches Ops Console.
2. Applicant application lifecycle:
   - Submit application.
   - View submitted status.
   - Block duplicate active application.
3. Admin review lifecycle:
   - Review an application.
   - Change status.
   - Confirm audit log.
4. Program lifecycle:
   - Create program.
   - Publish.
   - Verify applicant visibility.
   - Archive.
   - Verify it is no longer applicant-visible.
5. Tenant isolation:
   - Cross-tenant scoped read denial where fixtures allow it.
   - Realm role without `TenantMembership` does not grant authority.
6. Dashboard journey:
   - Dashboard pages load.
   - Task completion persists.
   - Notification read state persists.

## Assumptions

- This is local-development regression first, not cloud CI browser automation yet.
- Existing Vitest unit tests remain part of regression but are not treated as sufficient scenario coverage.
- Playwright is acceptable as a dev dependency for browser scenario expansion.
- Existing unrelated README cleanup work remains separate.
