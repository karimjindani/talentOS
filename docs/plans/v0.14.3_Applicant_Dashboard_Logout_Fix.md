# v0.14.3 Applicant Dashboard Logout Fix Plan

Code version: `v0.14.3`

## Summary

Fix two related logout defects (D-066):

1. **No logout from the accepted-applicant dashboard.** The dashboard layout replaces `PortalHeader`
   with `ApplicantShell`, which had no sign-out affordance. Because `v0.12.0` redirects accepted
   applicants from `/` and `/application` to `/dashboard`, an accepted applicant could never reach a
   page with a Logout button â€” they were trapped in an authenticated session.
2. **RP-initiated logout was broken on every tenant subdomain.** All logout forms built
   `post_logout_redirect_uri` from the request Host (`http://demo.lvh.me:3100/`), but Keycloak only
   supports `*` wildcards at the END of a redirect-URI pattern â€” the registered
   `http://*.lvh.me:3100/*` hostname wildcard never matches, so Keycloak answered
   "Invalid redirect uri" for any logout initiated from a tenant subdomain (latent since `v0.12.1`;
   the `v0.10.2` logout validation ran on the canonical host only).

## Key Changes

- `packages/auth-web/src/logout.ts`
  - `buildEndSessionUrl` gains an optional OIDC `state` parameter.
  - New `buildTenantLogoutUrl({ issuer, idToken, clientId, authUrl, requestOrigin })`: always sends
    `post_logout_redirect_uri` to the canonical AUTH_URL origin's `/logged-out` route (the only origin
    Keycloak can validate) and carries the tenant origin in `state`.
- New `/logged-out` route in both apps (`apps/applicant/app/logged-out/route.ts`,
  `apps/admin/app/logged-out/route.ts`): reads the echoed `state` and bounces the user back to their
  tenant via the existing allow-listed `resolveTenantRedirect` (canonical origin + `APP_BASE_DOMAIN`
  subdomains only â€” not an open redirect).
- Shared logout server actions: `apps/applicant/lib/logout-action.ts` and
  `apps/admin/lib/logout-action.ts` (`"use server"`), replacing the three duplicated inline forms in
  `PortalHeader`, the applicant `access-denied` page and the admin root layout.
- `apps/applicant/components/ApplicantShell.tsx`: Logout button added to the bottom sidebar user block
  (`<form action={logoutAction}>`).
- `apps/admin/middleware.ts`: `/logged-out` exempted from the authentication gate (the post-logout
  redirect arrives without a session by definition).

## Test Plan

- Unit tests:
  - `packages/auth-web/src/logout.test.ts`: `state` parameter emission; `buildTenantLogoutUrl` always
    targets canonical `/logged-out`, carries the tenant origin in `state`, normalizes AUTH_URL.
  - `apps/applicant/lib/logout-action.test.ts` (new): builder inputs (id_token, AUTH_URL, request
    origin), tenant-origin derivation from Host, sign-out-before-redirect ordering, missing-id_token
    path.
- Browser e2e (Playwright, real Keycloak + containers):
  - Accepted applicant: `/dashboard` shows sidebar Logout; clicking it terminates SSO and returns to
    `demo.lvh.me:3100`; revisiting `/dashboard` requires a fresh login (no silent re-login).
  - Admin: logout from `demo.lvh.me:3200` returns to the tenant and terminates SSO.
  - `/logged-out` guard: tenant subdomain allowed; foreign and look-alike hosts collapse to the
    canonical origin.
- Required validation: `npm.cmd run typecheck`, `npm.cmd run test`, `npm.cmd run build`,
  `npm.cmd run lint`, `docker compose up -d --build applicant admin`, browser e2e above.
