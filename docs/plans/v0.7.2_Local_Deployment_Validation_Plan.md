# v0.7.2 Local Deployment Validation Plan

Code version: `v0.7.2`

Base runtime commit: `2086e306e4483a7a9910b83cf0bc1d875c52eb08`

Date: `2026-06-30`

## Summary

Deploy the latest TalentOS baseline locally only. Alibaba Cloud deployment is out of scope for this
iteration.

The local deployment validates `v0.7.1` runtime behavior with Docker Compose:

- Applicant Portal
- Admin Portal
- PostgreSQL
- Keycloak
- Keycloak PostgreSQL
- MinIO
- MinIO setup job

## Local Runtime Configuration

Use local-safe ports:

- Applicant Portal: `3100`
- Admin Portal: `3200`
- Keycloak: `8080`
- MinIO API: `9000`
- MinIO Console: `9001`
- PostgreSQL: `55432`

Use a local `.env` file based on `.env.example`. The `.env` file remains ignored and must not be
committed.

## Deployment Steps

1. Pull latest GitHub `main`.
2. Refresh local `.env` with the current v0.7.1 environment variables.
3. Reset only TalentOS Docker containers and volumes with Docker Compose.
4. Rebuild and start the local stack.
5. Run Prisma migrations and seed data.
6. Validate public and protected routes.
7. Record results in `docs/testing/v0.7.2_Local_Deployment_Validation_Test_Results.md`.

## Assumptions

- Docker Desktop is running.
- Local TalentOS data is disposable for this validation.
- OpenPay containers and volumes must not be changed.
- Alibaba Cloud credentials and resources are not used.
