# v0.1.0 Initial Platform Architecture Plan

Code version: `v0.1.0`

Baseline commit: `4e2390ce270ef1e049652495885d792a0cbed959`

Plan status: Implemented

## Summary

Build TalentOS as a Dockerized, multi-tenant, white-label SaaS platform with two portals:

- Public Applicant Portal: tenant-branded landing page, applicant signup, guided 2FA setup, and application submission.
- Program Admin Portal: tenant owner/admin login, application review, basic program management, and audit visibility.

Use a scalable full-stack web architecture: Next.js, PostgreSQL, Prisma, custom credentials, TOTP 2FA, Docker Compose, and a stateless app container that can scale horizontally.

The first implementation is an Applications First vertical slice scaffold that establishes foundations for programs, missions, AI mentor, GitHub integration, portfolios, certificates, leaderboards, and hiring recommendations.

## Architecture Decisions

- Next.js TypeScript full-stack web application.
- PostgreSQL as the primary database.
- Prisma ORM for schema, migrations, and typed data access.
- Tailwind CSS for maintainable applicant/admin UI foundations.
- Docker Compose for local and VPS deployment.
- Shared database multi-tenancy using tenant-scoped records.
- Subdomain-based tenant resolution.
- Email/password authentication with authenticator-app TOTP 2FA guidance.
- Admin roles limited to `OWNER` and `ADMIN` in the first architecture.
- AI mentor represented by an auditable service boundary stub.

## Implementation Structure

- `apps/web`: Next.js web app, routes, middleware, UI components, and API boundaries.
- `packages/db`: Prisma schema, seed script, and database client.
- `packages/auth`: reusable auth, TOTP, tenant, role, and workflow utilities.
- `docs`: product, architecture, deployment, data, testing, plan, and version documentation.
- `docker-compose.yml`: local/VPS service definitions.
