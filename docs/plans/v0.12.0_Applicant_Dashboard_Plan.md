# Plan: v0.12.0 ‚Äî Applicant Dashboard

**Version:** `v0.12.0` ¬∑ **Decision:** D-059 ¬∑ **Previous:** `v0.11.4`

**Branch:** `feat/v0.12.0-applicant-dashboard`

## Objective

When an applicant's application is ACCEPTED, hide the "Apply" option and show a professional dashboard at `/dashboard`. The dashboard provides: 4-week program progress, notifications, due dates, video resources from HR/leads, assigned tasks, and a calendar.

## Scope

- **Applicant portal only** ‚Äî admin-side task/video management is deferred to a future iteration.
- **Full schema changes** ‚Äî 4 new Prisma models + 1 enum + 1 join table + migration.
- **External video URLs** (YouTube/Loom) ‚Äî no file upload needed.
- **Pre-defined tasks per program** ‚Äî tasks are created at the program level; admin editing is future.

## Implementation

### Phase 1: Schema & Migration
- New enum: `NotificationType` (INFO, WARNING, SUCCESS, TASK_DUE)
- New models: `ProgramTask`, `VideoResource`, `Notification`, `CalendarEvent`
- New join table: `UserTaskCompletion` (tracks which applicant completed which task)
- Relations added to `Tenant`, `User`, `Program`
- Migration: `20260703150655_v0_12_0_applicant_dashboard`
- DB helpers: `packages/db/src/dashboard.ts` (listProgramTasks, listTasksByWeek, listVideoResources, listCalendarEvents, listUserNotifications, countUnreadNotifications, markNotificationRead, createNotification, listCompletedTaskIds, markTaskCompleted, getApplicantProgramProgress)

### Phase 2: Dashboard Shell & Routing
- `apps/applicant/components/ApplicantShell.tsx` ‚Äî client component with sidebar (mirrors admin SidebarNav pattern), 7 nav items, unread notification badge, user info pinned to bottom
- `apps/applicant/middleware.ts` ‚Äî added `/dashboard` to protected prefixes
- `apps/applicant/app/dashboard/layout.tsx` ‚Äî wraps pages with shell, redirects to `/application` if no accepted application

### Phase 3: Dashboard Pages
- `/dashboard` ‚Äî overview: welcome header, quick stats (progress, tasks completed/pending, days remaining), 4-week progress bars, current tasks, recent notifications, upcoming events
- `/dashboard/program` ‚Äî program details + 4-week breakdown with per-week tasks/resources/progress
- `/dashboard/tasks` ‚Äî all tasks grouped by week with status (pending/completed/overdue) and due dates
- `/dashboard/resources` ‚Äî video resources grouped by week, YouTube embed or external link
- `/dashboard/calendar` ‚Äî upcoming and past events with date/time/location
- `/dashboard/notifications` ‚Äî full notification list with mark-as-read server action
- `/dashboard/profile` ‚Äî read-only applicant info + application details

### Phase 4: Post-Acceptance Routing
- Landing page (`/`) redirects to `/dashboard` if accepted application exists
- `/application` page redirects to `/dashboard` if accepted
- `PortalHeader` shows "Dashboard" link instead of "Apply" when accepted
- Login default callback remains `/application` (which redirects to dashboard if accepted)

### Phase 5: Seed Data & Tests
- `scripts/seed-dashboard.ts` ‚Äî creates demo tasks (9 tasks across 4 weeks), videos (5), calendar events (5), notifications (3)
- `packages/db/src/dashboard.test.ts` ‚Äî 24 tests covering all DB helpers with mocked Prisma
- `apps/applicant/components/ApplicantShell.test.ts` ‚Äî 10 tests covering nav active state and completeness
- Total regression suite: 125 tests (24 new), all passing

### Phase 6: SDLC Documentation
- All 10 SDLC docs updated (plan, test results, version baseline, architecture, data model, data dictionary, decision log, product backlog, testing strategy, deployment, sdlc)
