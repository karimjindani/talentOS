# v0.1.1 Local TalentOS Deployment Plan

Code version: `v0.1.1`

Previous baseline: `v0.1.0`

Previous baseline commit: `4e2390ce270ef1e049652495885d792a0cbed959`

Plan status: Implemented

## Summary

Run TalentOS locally with Docker Desktop while leaving existing OpenPay containers running.

OpenPay already uses local ports `3000` and `5432`, so TalentOS uses alternate host ports:

- Web: `http://localhost:3100`
- PostgreSQL: `localhost:55432`

## Implementation

- Make Docker Compose host ports configurable with `WEB_PORT` and `POSTGRES_PORT`.
- Add local `.env` values for the alternate ports.
- Keep the app container listening internally on port `3000`.
- Keep PostgreSQL listening internally on port `5432`.
- Expose the services on host ports `3100` and `55432`.
- Bump package versions to `0.1.1`.

## Deployment Commands

```powershell
npm.cmd install
npm.cmd run db:generate
docker compose up postgres -d
npm.cmd run db:migrate
npm.cmd run db:seed
docker compose up web -d --build
```

- Admin applications: `http://localhost:3100/admin/applications`

## Assumptions

- Existing OpenPay containers remain running.
- Local `.env` is not committed because it can contain secrets.
- `.env.example` documents default ports, while local `.env` can override them.
