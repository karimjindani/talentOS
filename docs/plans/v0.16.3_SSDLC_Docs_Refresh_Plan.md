# v0.16.3 — SSDLC Documentation Audit and Refresh Plan (D-071)

Code version: `v0.16.3` (documentation-only patch)

## Context

Follow-on to the `v0.16.2` vision audit (D-070). Eight SSDLC docs were flagged as release-stale:
`Architecture.md`, `CI_CD_Pipeline.md`, `Data_Dictionary.md`, `Data_Model.md`, `Deployment.md`,
`Regression_Scenarios.md`, `Source_Control_Policy.md`, `Testing_Strategy.md`. An audit against
committed code (code baseline `v0.16.1`; the Prisma schema has been frozen since the `v0.15.0`
migration — `v0.16.0`/`v0.16.1`/`v0.16.2` were explicitly no-schema-change) confirmed staleness in
all eight, ranging from header-only lag to operationally dangerous gaps. This iteration refreshes
all of them (full refresh, not just critical fixes) as one docs-only baseline.

## Audit findings (ranked by real drift)

### 1. `Deployment.md` — declared `v0.12.2`, four minor baselines stale (worst)

- Migration changelog stopped at `v0.12.0`, omitting the two **required** shipped migrations
  `20260704160000_v0_14_0_mission_engine_mvp` (`v0.14.0`) and
  `20260706090000_v0_15_0_mission_submissions` (`v0.15.0`) — a deployer following the doc would not
  know about the mission-engine and submissions schema changes.
- Local Validation URLs omitted admin `/missions*`, `/operations`, `/organizations`,
  `/programs/[id]/content` and the submission-review route; applicant `/dashboard/missions`,
  `/access-denied`, `/logged-out`.
- Smoke tests stopped at `v0.7.3` (CV checks); no mission/submission/dashboard-progress coverage.
- Still accurate: `local:bootstrap`/`doctor`/`smoke-login`, hostnames, Compose topology,
  Keycloak/MinIO sections; `worker` correctly future.

### 2. `Data_Model.md` + `Data_Dictionary.md` — both declared `v0.15.0`; body gaps are older

- The five `v0.12.0` dashboard models (`ProgramTask`, `VideoResource`, `Notification`,
  `CalendarEvent`, `UserTaskCompletion`) existed only in changelog prose — absent from the Core
  Entities list, the ER diagram, and the Data Dictionary field tables. `NotificationType` likewise.
- The ER diagram showed 12 of 20 models; it also omitted `PortfolioArtifact`/`Certificate` while
  inconsistently drawing `KnowledgeBaseDocument`/`AIInteraction`, and missed the
  `Tenant→logoFile`, notification, task-completion and program-content relations.
- Field-level gaps: `Tenant.logoFileId` and `User.lastLoginAt` missing from the field tables.
- The four stub models were framed as "Future-Ready" although they are migrated real tables
  (app-unused), and were entirely absent from the Data Dictionary.
- Both docs carried a stale `v0.14.0` baseline-commit placeholder; migrations `20260627084605_init`
  and `20260630080000_regression_data_markers` were never cited.
- Strongest part: the Mission and Submission field tables were complete and type-accurate.

### 3. `Testing_Strategy.md` — declared `v0.14.1`, six releases stale

- Latest stated unit count **146; actual 202** (30 files). Latest scenario state 15 (`v0.13.0`);
  **actual 22** in `scripts/regression/run.ts`.
- Entirely missing: `v0.15.0` submissions tests (`submissions.test.ts`, 20 tests), `v0.16.0`
  program-content tests (6), the mission-progress scenario, and the `v0.16.1` Playwright capture.
- The storage section presented presign round-trip checks without noting the `storage` scenario
  area is a documented skip.

### 4. `Regression_Scenarios.md` — declared `v0.16.0`, but with older gaps

- The matrix omitted three real `v0.15.0` scenarios: the full submission loop, the reviewer role
  matrix, and cross-tenant submission-read denial.
- The marker entity list omitted `Submission`.
- "Known Gaps After `v0.14.0`" heading was never advanced.

### 5. `Architecture.md` — declared `v0.16.0`, substantively near-current

- The shared-package sentence omitted `packages/storage` (listed elsewhere in the same doc).
- `apps/ops` was never described as an application ("two portals" framing) — the host-run,
  Keycloak-gated Ops Console on `127.0.0.1:3300` is a third app, intentionally not containerized.
- The portal-layout diagram omitted `/missions/new`, `/missions/[id]`, the `v0.15.0`
  submission-review page, `/programs/[id]`, `/programs/[id]/content`, `/dashboard/missions[/[id]]`
  and both `/logged-out` routes (`v0.14.3`).
- Two stale `v0.3.1` labels for work that either shipped in `v0.11.0` (auto-provisioning) or is
  plain future work (Users/Roles UI).

### 6. `CI_CD_Pipeline.md` + `Source_Control_Policy.md` — declared `v0.11.2`, header-stale only

- The CI stage table matches `.github/workflows/ci.yml` exactly; all CD/security-scan content is
  correctly labeled future and confirmed absent.
- `CI_CD_Pipeline.md` header said `v0.11.2` while describing the `realm-import` job "added in
  `v0.11.3`"; the merge-gate description in `Source_Control_Policy.md` omitted `realm-import`.

### Bonus finding

`Version_Baseline.md` "Portal Scope" and "Package Scope" were `v0.3.0` snapshots omitting nearly
everything shipped since — including `apps/ops` and `packages/storage`.

## Changes

All eight docs (plus `Version_Baseline.md`) updated as described above under one
`Code version: v0.16.3` header each; bookkeeping adds this plan,
`docs/testing/v0.16.3_SSDLC_Docs_Refresh_Test_Results.md`, Decision Log entry `D-071` and the new
current baseline in `docs/Version_Baseline.md`.
