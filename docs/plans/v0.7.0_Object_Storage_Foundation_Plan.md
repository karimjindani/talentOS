# v0.7.0 Object Storage Foundation Plan

Code version: `v0.7.0`

Previous baseline: `v0.6.0`

Previous baseline commit: `8b0e11b`

Plan status: Implemented

## Summary

Adds the reusable **object-storage foundation** so later slices (CV-on-apply, program materials,
certificates, portfolio assets) can store binaries. Before `v0.7.0` the platform had no file storage —
only structured text in PostgreSQL and external URL strings. The object store is **MinIO**, self-hosted
in every environment (local and the Alibaba ECS cloud box) as a Docker Compose service alongside
Postgres/Keycloak; it is accessed through the S3 API, so the code is provider-neutral. This is the first
schema change since `v0.3.0` (a new `StoredFile` model + migration).

## Decisions

- **MinIO** as the object store, accessed via the AWS SDK v3 (`@aws-sdk/client-s3` +
  `s3-request-presigner`) with `forcePathStyle=true`. Same image/config local and cloud.
- **Direct browser ↔ MinIO** transfer via **presigned URLs** (PUT upload, GET download, short TTL); Next
  never proxies bytes.
- **Endpoint** mirrors the Keycloak pattern (`S3_ENDPOINT=http://host.docker.internal:9000` locally) so
  presigned URLs resolve in both the browser and the containers.
- **Tenant-namespaced keys** `tenant/{tenantId}/{category}/{uuid}-{safeName}`; the bucket is **private**
  (no anonymous policy); downloads only via presigned GET.
- File **metadata in PostgreSQL** (`StoredFile`) → tenant-scoped and auditable; MinIO holds only bytes.
- Credentials env-only (consistent with `D-039`).

## Implementation

- **Infra (`docker-compose.yml`, `.env.example`)** — `minio` service (API `9000`, console `9001`,
  private data volume, healthcheck) + one-shot `minio-setup` (`minio/mc`) that creates the private
  bucket. Admin service gets `S3_*` env; `.env.example` documents the keys with MinIO dev defaults.
- **Storage package (`packages/storage`, `@talentos/storage`)** — `client.ts` (S3 client from env),
  `index.ts` (`getPresignedUploadUrl`, `getPresignedDownloadUrl`, `deleteObject`), `keys.ts` (pure
  `sanitizeFilename`, `buildObjectKey`), unit-tested in `keys.test.ts`.
- **Schema + migration** — `StoredFile` model (`tenantId`, `ownerUserId?`, `bucket`, `storageKey @unique`,
  `originalName`, `contentType`, `size`, `category`, `status PENDING|READY`) with Tenant/User
  back-relations; migration `20260629101218_object_storage`.
- **DB helpers (`packages/db/src/files.ts`)** — `createStoredFile` (+`file.created` audit),
  `markStoredFileReady`, `getStoredFile`, `listStoredFiles`, `deleteStoredFile` (+`file.deleted` audit).
- **Reference endpoints (`apps/admin/app/api/files/`)** — `POST presign-upload` (auth + tenant +
  content-type allowlist + 10 MB cap → `StoredFile` PENDING + presigned PUT), `POST [id]/confirm`
  (→ READY), `GET [id]/download` (tenant-scoped → 302 to presigned GET). Reusable later by the applicant
  app for CV-on-apply.
- **Wiring** — `@talentos/storage` added to admin deps, `transpilePackages`, app + root tsconfig paths,
  and the `Dockerfile` `deps` stage.
