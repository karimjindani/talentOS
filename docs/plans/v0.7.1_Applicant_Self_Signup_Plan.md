# v0.7.1 Applicant Self-Signup Plan

Code version: `v0.7.1`

Previous baseline: `v0.7.0`

Previous baseline commit: `7d73552`

Plan status: Implemented

## Summary

Adds applicant self-signup. Since `v0.3.0` Keycloak owns signup, but realm self-registration was never
enabled and the applicant portal had no signup entry point, so there was no way for a new applicant to
create an account (only pre-provisioned/seeded users could sign in). This patch enables Keycloak
self-registration and surfaces a "Create account" entry in the applicant portal. It is a small,
non-breaking addition (patch `v0.7.1`); no code/schema baseline change beyond config + UI.

## Decisions

- Keep the `v0.7.0` baseline; record this as patch `v0.7.1` (Version_Baseline convention).
- **Keycloak owns signup** (consistent with `D-031`): enable realm `registrationAllowed` rather than
  building a custom signup form (the custom Admin-REST approach stays with the deferred `v0.3.1` IAM UI).
- New registrants default to the **APPLICANT** realm role (`defaultRoles`), and username = email
  (`registrationEmailAsUsername`), matching the email-keyed `User` model and `provisionApplicantUser`.

## Operational note (realm re-import)

Keycloak imports the realm only when it does not already exist (`start-dev --import-realm`). Because the
realm is already persisted in `talentos-keycloak-postgres`, applying this change locally requires either
toggling the setting in the Keycloak admin console **or** recreating the Keycloak DB volume so the realm
re-imports (demo users are re-seeded from the JSON). On a fresh environment the import applies directly.
