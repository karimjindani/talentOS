# v0.16.4 SSDLC Compliance Audit Plan

## Summary

Audit the current `main` branch against `docs/sdlc.md` and produce a documented evidence matrix. This is
an audit-only baseline: no product code, schema, Docker or runtime behavior changes.

## Audit Scope

- Assess SSDLC principles 0–7.
- Verify versioned plans, test results, baselines and decision logs.
- Compare architecture, deployment, data model, data dictionary, testing and security docs against the
  current codebase.
- Run the required local validation commands.
- Identify gaps as documentation, code, process or local-environment issues.

## Validation Commands

- `git status --short`
- `git branch --show-current`
- `git log --oneline -20`
- `git diff --check`
- `npm.cmd run db:generate`
- `npm.cmd run db:migrate`
- `npm.cmd run test`
- `npm.cmd run typecheck`
- `npm.cmd run lint`
- `npm.cmd run build`
- `docker compose config --quiet`
- `npm.cmd run local:doctor`
- `npm.cmd run regression:all`
