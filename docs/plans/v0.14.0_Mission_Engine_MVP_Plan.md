# v0.14.0 Mission Engine MVP Plan

## Summary

Implement the Mission Engine MVP as the next TalentOS product slice. Admins can create, edit, publish,
archive and view missions for programs. Accepted applicants can view published missions inside their
dashboard. The demo seed includes a realistic Week 1 mission based on the Spiral Engineering Method.

## Key Changes

- Add Admin Mission Management:
  - `/missions`
  - `/missions/new`
  - `/missions/[id]`
- Extend `Mission` with:
  - `MissionStatus`: `DRAFT`, `PUBLISHED`, `ARCHIVED`
  - `weekNumber`, `order`
  - `objective`, `acceptanceCriteria`, `deliverables`, `evaluationCriteria`, `competencyTags`
- Add applicant dashboard mission routes:
  - `/dashboard/missions`
  - `/dashboard/missions/[id]`
- Add `manageMissions` capability for `SUPER_ADMIN` and `ORG_ADMIN`.
- Add audit events:
  - `mission.created`
  - `mission.updated`
  - `mission.status_changed`
- Seed the demo program with the Week 1 mission: `Build a Public Product Landing Page`.
- Add `regression:missions` and include it in `regression:all`.

## Test Plan

- Unit tests:
  - Mission status transitions.
  - `manageMissions` permission.
  - Tenant-scoped mission reads/writes.
  - Published-only applicant visibility.
  - Mission ordering by week/order.
- Scenario regression:
  - Mission publish/archive visibility.
  - Mission mutation permission denial for HR, Tech Lead and Applicant.
- Required validation:
  - `npm.cmd run db:generate`
  - `npm.cmd run typecheck`
  - `npm.cmd run test`
  - `npm.cmd run build`
  - `npm.cmd run lint`
  - `npm.cmd run local:bootstrap`
  - `npm.cmd run regression:missions`
  - `npm.cmd run regression:all`
  - `npm.cmd run ops:cleanup-regression`
  - `docker compose config --quiet`
  - `git diff --check`

## Assumptions

- Version is `v0.14.0`.
- Submission workflow is deferred to `v0.15.0`.
- Engineering Journal is deferred until after Submission Engine.
- Public Portfolio is deferred until missions and submissions exist.
- Mission content is stored as structured text/markdown fields, not a separate CMS.
- Applicant mission access is limited to accepted applicants in the relevant program.
