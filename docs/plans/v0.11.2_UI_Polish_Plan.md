# v0.11.2 — UI Polish (Apply Page, Admin Sidebar, Review Back Button)

Code version: `v0.11.2` · Previous: `v0.11.1` · Date: `2026-07-03`

## Problem

Three UX gaps in the applicant and admin portals:

1. **Applicant Apply page** (`/apply`) — the form was plain and unprofessional: flat inputs with no
   focus rings, no visual hierarchy, no section grouping, a plain submit button with no hover state.
2. **Admin Portal sidebar** — nav links had hover styles but no **active-state indicator**. After
   clicking a link, there was no visual feedback of which page was active — the user couldn't tell
   which section they were on.
3. **Admin Application Review page** (`/applications/[id]`) — no **back button** to return to the
   applications list. Admins had to use the sidebar or browser back button.

## Fix (UI-only, no schema or logic change)

### 1. Apply page professionalization (`apps/applicant/app/apply/page.tsx`)

Replaced the plain form render with a professional, branded, card-based layout using existing Tailwind
brand tokens (`brand-navy`, `brand-blue`, `brand-mist`):

- **Page header banner**: `bg-brand-mist` rounded banner with 📝 icon in a `bg-brand-blue` circle.
- **Form card**: `rounded-2xl border border-slate-200 bg-white p-8 shadow-md` with `space-y-8` spacing.
- **Applicant info chip**: avatar circle with initial, name and email in `bg-slate-50` bar.
- **Section dividers**: uppercase `text-brand-blue` labels with `h-px bg-slate-200` lines.
- **Three sections**: "Program & Motivation", "Documents" (dashed-border upload zone), "Profile Links
  (optional)".
- **All inputs**: consistent `rounded-lg border-slate-300` with `focus:ring-2 focus:ring-brand-blue/20`.
- **Submit button**: full-width `rounded-xl bg-brand-blue` with `hover:bg-brand-navy`.
- **Error banners**: `rounded-xl border-amber-200 bg-amber-50` with ⚠️ icon.

Server action `submitApplication` is **unchanged** — only the JSX render section was replaced.

### 2. Admin sidebar active-state indicator (`apps/admin/components/SidebarNav.tsx`, NEW)

Extracted the inline `<nav>` from `apps/admin/app/layout.tsx` into a new client component
`SidebarNav.tsx` that uses `usePathname()` from `next/navigation` to detect the current route:

- **Active style**: `bg-brand-blue text-white font-semibold shadow-sm` — high contrast, clearly
  distinguishable from the hover state.
- **Inactive style**: keeps the existing `text-slate-700 hover:bg-slate-100 hover:text-brand-blue`.
- **Route matching**: exact match for `/` (Overview), `startsWith` for all others (so
  `/applications/123` highlights "Applications").
- **All admin roles** see the active state (ORG_ADMIN, HR, TECH_LEAD, SUPER_ADMIN). Only the
  "Organizations" link remains SUPER_ADMIN-only.
- The `isActive` function and `NAV_ITEMS` array are exported for unit testing.

### 3. Review page back button (`apps/admin/app/applications/[id]/page.tsx`)

Added a "← Back to Applications" link at the top of the review page, before the `<h1>`:

- Styled as `inline-flex items-center gap-1.5 text-sm font-medium text-brand-blue hover:text-brand-navy`.
- Links to `/applications`.
