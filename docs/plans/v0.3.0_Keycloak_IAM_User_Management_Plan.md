## Summary

Establishes Keycloak as the TalentOS identity provider and wires real authentication and role-based
authorization into both isolated portals, following the `codex.md` "Keycloak IAM + Admin User Management"
baseline. The slice is **staged**: `v0.3.0` delivers the IAM + RBAC foundation; the Admin Portal
user/org/role-management UI (Keycloak Admin REST API CRUD) is `v0.3.1`.

## Role model

| Role | Scope | Keycloak | TalentOS DB |
| --- | --- | --- | --- |
| `SUPER_ADMIN` | platform | realm role | `User.platformRole` |
| `ORG_ADMIN` / `HR` / `TECH_LEAD` / `APPLICANT` | organization | realm roles | `TenantMembership.role` |

Authorization is a capability matrix (from `codex.md` §3) encoded in `packages/auth/src/permissions.ts`
and unit tested. Admin-portal access = SUPER_ADMIN or ORG_ADMIN/HR/TECH_LEAD; APPLICANT denied.

Org/role mapping = Keycloak realm roles for identity + the TalentOS DB (`Tenant`/`TenantMembership`) for
org scoping.

## Implementation

- **Keycloak infra:** `keycloak` + dedicated `keycloak-postgres` services in `docker-compose.yml`; realm
  import `keycloak/import/talentos-realm.json` with the 5 roles, password policy
  (`length(12)`, upper/lower/digit/special, notUsername, passwordHistory(5)), TOTP OTP policy, the
  `talentos-applicant` / `talentos-admin` OIDC clients, a seeded Super Admin (temporary password +
  `UPDATE_PASSWORD`/`CONFIGURE_TOTP` first-login actions) and demo org users.
- **RBAC:** `packages/auth/src/rbac.ts` (new `TenantRole`/`PlatformRole`, `canAccessAdminPortal`,
  `isSuperAdmin`, `assertTenantScopedAccess`) + `permissions.ts` capability matrix; tests updated/extended.
- **Auth library:** `packages/auth-web` (`@talentos/auth-web`) — `createTalentosAuth()` (NextAuth v5 +
  Keycloak OIDC, JWT sessions, edge-safe realm-role decoding), `canEnterAdminPortal`, role helpers.
- **App wiring:** each app gets `auth.ts`, `app/api/auth/[...nextauth]/route.ts`, an `auth`-wrapped
  `middleware.ts`, `SessionProvider`, and a session-aware header. Applicant `/login` uses a Keycloak
  sign-in button and protects `/application`; placeholder `signup`/`2fa` pages removed. Admin gates every
  route on an admin-capable role, redirecting denied users to a new `/forbidden` page.
- **DB:** `User.keycloakSubjectId` (unique), `emailVerified`, optional `passwordHash`, `platformRole`;
  new `PlatformRole` enum; `TenantRole` → ORG_ADMIN/HR/TECH_LEAD/APPLICANT; migration
  `20260628000000_keycloak_iam_rbac`; seed creates the Super Admin + one demo user per org role.

## Issuer / hostname

Browser and app containers use one issuer `http://host.docker.internal:8080/realms/talentos` (Docker
Desktop maps `host.docker.internal` on both sides), avoiding the OIDC `iss` mismatch. Apps get
`extra_hosts: host.docker.internal:host-gateway`. Local `npm run dev` uses `http://localhost:8080/...`.
