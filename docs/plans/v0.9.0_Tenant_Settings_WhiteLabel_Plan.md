# v0.9.0 — Tenant Settings / White-label Configuration — Implementation Plan

Code version: `v0.9.0`

## Objective

Make white-labelling real: an ORG_ADMIN-gated Settings page to configure organisation name, brand
colors and logo, persisted with audit log, and applied live to both portals via CSS custom properties.

## Decisions

See `Decision_Log.md` D-047 for full rationale. Summary:
- CSS vars in `tailwind-preset.ts` with hex fallbacks — no component changes needed
- Logo upload: server action + `putObject` (no presign/confirm flow)
- `Tenant.logoFileId` FK to `StoredFile` (migration `20260701120000_tenant_logo_file_id`)
- Public logo route on applicant app: unauthenticated, host-resolved, IDOR-safe

## Security Controls

- Colors validated with `^#[0-9a-fA-F]{6}$` server-side before persistence and before CSS injection
- Logo allowlist: PNG/JPEG/WebP only (no SVG — XSS vector), max 2 MB
- Public logo route is IDOR-safe: resolves file from host-tenant only via `getStoredFile(id, tenant.id)`
- `manageTenantSettings` checked in both page (UI gating) and server action (enforce)
- All writes are audited as `tenant.branding_updated`
