# v0.12.2 Local Deployment Hardening Plan

## Summary

Fix the local deployment experience so a developer can run one bootstrap command, open all portals and
complete login without `host.docker.internal` browser redirects, stale Keycloak clients, issuer mismatches
or dashboard seed confusion.

## Key Changes

- Use `http://keycloak.lvh.me:8080/realms/talentos` as the local Keycloak issuer for both browser and
  app containers.
- Use `http://minio.lvh.me:9000` as the local browser/container MinIO API endpoint.
- Add local commands:
  - `npm.cmd run local:bootstrap`
  - `npm.cmd run local:doctor`
  - `npm.cmd run local:smoke-login`
- Repair existing local Keycloak realms non-destructively during bootstrap.
- Add `accepted@demo.talentos.local` as a seeded accepted applicant for applicant-dashboard validation.
- Make `scripts/seed-dashboard.ts` idempotent.
- Update README, deployment, architecture, decision log and version baseline docs for `v0.12.2`.

## Assumptions

- Local Docker volumes are preserved by default.
- `lvh.me`, `*.lvh.me`, `keycloak.lvh.me` and `minio.lvh.me` are acceptable local-development
  hostnames.
- This is a local developer-experience fix; production deployment topology is unchanged.
