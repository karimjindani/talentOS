# v0.8.0 Operations Page Plan

Code version: `v0.8.0`

Base commit before implementation: `0906ac85c43b84bc22fdedd40c0b3a808ea112e3`

Date: `2026-06-30`

## Summary

Add a local-development Operations page to the Admin Portal at `/operations`.

The page is a guided dashboard, not a host-control panel. It shows live app-visible health checks and
copyable commands for regression tests, regression data cleanup and local reset. The Admin app does not
run Docker or npm commands on the host.

## Key Changes

- Add an Admin Portal `Operations` navigation item and page.
- Add `GET /api/operations/health` for app-visible health checks.
- Restrict the Operations page/API to `SUPER_ADMIN` and `ORG_ADMIN`.
- Add `RegressionDataMarker` so cleanup can target only explicitly marked regression data.
- Add `npm.cmd run ops:cleanup-regression` for safe marker-based cleanup.

## Safety Rules

- No Docker socket or host shell access is exposed to the Admin app.
- Reset is shown as copyable local commands only.
- Regression cleanup only deletes rows referenced by `regression_data_markers`.
- Unmarked seeded and user-created data is never deleted by regression cleanup.

## Test Plan

- Unit tests for health summary and operations commands.
- Unit tests for regression cleanup ordering and safety boundaries.
- Full regression, typecheck, build, lint, Docker Compose config and whitespace checks.
