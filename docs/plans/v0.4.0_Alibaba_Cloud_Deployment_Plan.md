# v0.4.0 Alibaba Cloud Deployment Plan

Code version: `v0.4.0`

Base commit before implementation: `44e9b12d1f9225c58526b609e8c75686fb005317`

Date: `2026-06-28`

## Summary

Deploy TalentOS to Alibaba Cloud Singapore on a single ECS instance using Docker Compose.

This is the first cloud validation deployment for the current platform baseline. It keeps the existing
runtime shape:

- Applicant Portal container
- Admin Portal container
- TalentOS PostgreSQL container
- Keycloak container
- Keycloak PostgreSQL container

## Deployment Target

- Cloud provider: Alibaba Cloud
- Region: Singapore (`ap-southeast-1`)
- Initial shape: single ECS instance
- Public access model: public IP and explicit validation ports
- Runtime: Docker Compose
- Database: PostgreSQL container on the ECS instance for this validation baseline

## Credential Handling

RAM credentials are supplied from a local file outside the repository.

Credential rules:

- Do not paste secrets into chat.
- Do not store RAM keys inside the repository.
- Do not commit `.env`, `.env.production`, AccessKeys or cloud-generated private keys.
- Load RAM credentials only into the active shell session.
- Rotate or disable temporary RAM keys after deployment if they were created only for this deployment.

## Alibaba Cloud Resources

Create or configure:

- ECS instance in `ap-southeast-1`.
- Public IPv4 address.
- Security group for TalentOS validation traffic.
- SSH access for deployment.

Initial security group inbound rules:

- `22/tcp` restricted to the deployer's current public IP.
- `3100/tcp` for Applicant Portal validation.
- `3200/tcp` for Admin Portal validation.
- `8080/tcp` for Keycloak validation.

PostgreSQL ports must not be publicly exposed.

## Deployment Steps

1. Verify the local repository is clean and aligned with GitHub `main`.
2. Load RAM credentials from the local secret file without printing them.
3. Verify Alibaba Cloud identity/credential access.
4. Provision or identify the ECS instance.
5. Configure security group rules.
6. Install Docker and Docker Compose on ECS.
7. Clone `karimjindani/talentOS` on ECS.
8. Create server-only production environment values.
9. Start services with Docker Compose.
10. Run database migration and seed steps where required.
11. Validate the public URLs.
12. Record test evidence in `docs/testing/v0.4.0_Alibaba_Cloud_Deployment_Test_Results.md`.

## Validation URLs

Replace `<ECS_PUBLIC_IP>` with the assigned Alibaba Cloud public IP:

- Applicant Portal: `http://<ECS_PUBLIC_IP>:3100`
- Admin Portal: `http://<ECS_PUBLIC_IP>:3200`
- Keycloak: `http://<ECS_PUBLIC_IP>:8080`

## Follow-Up Hardening

After public-IP validation:

- Add domain names and HTTPS.
- Replace Keycloak `start-dev` with production startup configuration.
- Move PostgreSQL to Alibaba Cloud RDS for production readiness.
- Add backups, monitoring, and centralized logs.
- Restrict Keycloak admin access.
- Close direct validation ports after reverse proxy/domain routing is available.
