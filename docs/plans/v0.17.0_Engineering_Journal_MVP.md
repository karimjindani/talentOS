# v0.17.0 Engineering Journal MVP

Code version: `v0.17.0`

## Summary

Implement the first dedicated Engineering Journal module in the Applicant Portal. The journal is a
daily structured reflection system for accepted applicants and is separate from the older inline
`Submission.journalMarkdown` field used during mission submission review.

## Scope

- Add `EngineeringJournalEntry`, linked to tenant, applicant, program, mission and derived week number.
- Add `User.preferredJournalLanguage`.
- Add applicant dashboard navigation and pages for listing, creating and editing journal entries.
- Add a profile setting for preferred journal language.
- Add tenant-scoped, applicant-owned database helpers with mission/program validation.
- Add nullable AI review/scoring fields as placeholders only.

## Out Of Scope

- Voice-to-journal entry.
- Mobile logging.
- AI-generated weekly reflection.
- Real AI scoring.
- Competency mapping.
- Recruiter-visible portfolio pages.
- Screenshot or file uploads.
- Blocking mission submission until a journal entry exists.
- Auth, Keycloak, workflow or permission matrix changes.

## Security And Data Rules

- Journal entries are visible and editable only by the owning applicant.
- Writes require applicant portal tenant access.
- The selected mission must be published and belong to the applicant's accepted program.
- `weekNumber` is derived from the selected mission, not trusted from the browser.
- Back-dated entries are allowed, but each applicant can have only one entry per calendar date within
  a tenant.
- All journal writes record `journal.created` or `journal.updated` audit events.
