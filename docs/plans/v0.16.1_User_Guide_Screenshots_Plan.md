# v0.16.1 — Illustrated User Guide with Automated Screenshot Capture — Plan

Code version: `v0.16.1` (documentation/tooling patch on the `v0.16.0` baseline, commit `a95a42b`)

Date: `2026-07-07`

## Goal

Produce an illustrated end-user guide covering every user-facing flow of TalentOS — one full-page
screenshot per test case — and make the capture repeatable so the guide can be regenerated after
each iteration instead of rotting.

## Scope

1. **`scripts/user-guide/capture-screenshots.ts`** — a Playwright (Chromium) script that drives the
   running local Docker deployment through the real Keycloak OIDC flows and captures full-page
   PNGs at a 1440×900 viewport:
   - *Anonymous*: tenant landing page, portal login, Keycloak sign-in.
   - *New applicant*: Keycloak self-registration, `/apply` form (filled, PDF CV attached via a
     generated minimal PDF), submit, `/application` status page. A fresh
     `guide.applicant.<id>@demo.talentos.local` account is registered each run because existing
     tenant members are redirected away from `/apply`.
   - *Accepted applicant* (`accepted@demo.talentos.local`): dashboard overview and all dashboard
     sub-pages (missions + detail, tasks, resources, calendar, notifications, profile, program).
   - *Org Admin* (`orgadmin@demo.talentos.local`): admin overview, applications queue + review
     detail, programs + detail + content page (v0.16.0), missions + detail, settings, operations.
   - *Ops Console* (`127.0.0.1:3300`) signed in as Org Admin.
   - CLI section filter (`public`, `apply`, `dashboard`, `admin`, `ops`) for partial re-captures;
     detail pages are located from list-page links and skipped with a warning when absent.
2. **`docs/user-guide/screenshots/`** — 26 captured PNGs.
3. **`docs/user-guide/User_Guide.md`** — the guide: coverage map table linking each screenshot to
   its flow and regression test area (`npm run regression:<area>`), sections per journey
   (getting started / applicant journey / dashboard / admin portal / ops console), and
   regeneration instructions.

## Non-goals

- No product code, schema or configuration changes.
- The admin submission-review page (`/missions/{id}/submissions/{submissionId}`) is not captured:
  the demo mission had no reviewable submission link at capture time. Documented in the guide;
  re-running the capture after an applicant mission submission will include it.
- Screenshots are not wired into CI; capture is run manually against the local stack.

## Risks / considerations

- Each `apply`-section run creates one Keycloak user and one SUBMITTED application in the `demo`
  tenant (visible in the admin queue). Accepted as demo data; can be removed with the existing
  cleanup tooling if it becomes noise.
- Playwright's default browser directory may not be writable on managed Windows machines; the
  script header and guide document the `PLAYWRIGHT_BROWSERS_PATH` workaround.
- `scripts/` is outside the root `tsconfig.json` include set, so the capture script is executed by
  `tsx` (like the other repo scripts) and not covered by `npm run typecheck`.

## Acceptance

- All targeted pages captured (26 PNGs, one skip documented).
- Guide renders with correct relative image links.
- `npm test` remains green (no product code touched).
