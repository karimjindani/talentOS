# v0.15.1 — Four-Week Mission Seed (D-068)

Code version: `v0.15.1`

Decision: `D-068`

Type: patch (seed/demo content + cosmetic rendering fix). No schema change, no API change.

## Problem

The demo tenant seeds only the Week 1 mission ("Build a Public Product Landing Page"), while the
program TalentOS exists to deliver is a **four-week** AI-Native Software Engineering Apprenticeship
(`docs/curriculum.md`). Out of the box the platform cannot demonstrate the core promise of the
Spiral Engineering Method — the same lifecycle repeated at increasing complexity — because Weeks
2–4 do not exist as missions.

Separately, the mission pages shipped in `v0.14.0` render the week/difficulty separator as mojibake
(`Â€¢` instead of `•`) because the source files contain a double-encoded UTF-8 bullet.

## Sources of truth

The mission content is derived directly from the program documents:

- `docs/SEM.md` — every mission embeds the full 10-step lifecycle (Discover → Production Readiness
  Review); "Do everything every week. Increase complexity every week."
- `docs/curriculum.md` — weekly themes, skills, deliverables and the competency-coverage matrix.
- `docs/Competency_Framework.md` — `competencyTags` use the framework's competency names verbatim.
- `docs/Graduate_Profile.md` — missions target independent engineers; no peer code review anywhere.
- `docs/vision.md` / `docs/Product_Vision.md` — one continuous product; deployment progression
  static hosting → full-stack hosting → Docker + CI/CD → VPS with reverse proxy and SSL.

## Design

One continuous product — **TaskPilot**, the fictional AI-assisted task planner introduced in the
Week 1 mission — is taken from idea to production across four missions:

| Week | Mission | Difficulty | Theme (curriculum) |
| --- | --- | --- | --- |
| 1 | Build a Public Product Landing Page | BEGINNER | Build Something Real |
| 2 | Design and Build the TaskPilot Application | INTERMEDIATE | Build Like an Engineer |
| 3 | Containerize, Automate and Load-Test TaskPilot | ADVANCED | Build Like a Production Team |
| 4 | Take TaskPilot to Production | EXPERT | Build Like a Production Engineer |

Each mission carries: `objective`, a `brief` with business context plus the tailored 10-step SEM
lifecycle, `deliverables` (the curriculum's required deliverables for that week), measurable
`acceptanceCriteria`, Bronze/Silver/Gold/Platinum `evaluationCriteria`, and `competencyTags`.
Week 4 introduces the `Production Readiness` tag, matching the competency matrix (PRR reaches
Mastery only in Week 4).

## Changes

1. `packages/db/prisma/seed.ts` — refactored the single verbose Week 1 upsert into a data-driven
   `missionSeeds` array (Week 1 content byte-identical) and added Weeks 2–4. Upserts keyed on
   stable ids (`seed-mission-week-{n}-…`) keep the seed idempotent.
2. Mojibake fix — `apps/applicant/app/dashboard/missions/page.tsx`,
   `apps/applicant/app/dashboard/missions/[id]/page.tsx`, `apps/admin/app/missions/[id]/page.tsx`
   now render the separator via the encoding-proof JSX escape `{"•"}`.

No schema, capability, route or workflow changes.
