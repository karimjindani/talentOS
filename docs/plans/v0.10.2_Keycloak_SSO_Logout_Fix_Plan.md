# v0.10.2 — Keycloak SSO Logout Fix (RP-Initiated Logout) — Implementation Plan

Code version: `v0.10.2`

## Objective

Fix logout not being effective: after clicking Logout, refreshing the page silently re-authenticated
the user. The app cleared only its own NextAuth cookie while the Keycloak SSO session stayed alive.

## Root cause

`createTalentosAuth` (`packages/auth-web/src/auth.ts`) uses JWT sessions; both logout actions called
NextAuth `signOut()` only, which clears the app cookie but never terminates the Keycloak
`KEYCLOAK_IDENTITY` SSO session. The config also never stored the `id_token`, so there was no
`id_token_hint` available for a proper logout.

## Fix — OIDC RP-initiated logout

- Persist `account.id_token` on the JWT and expose it on the session (`packages/auth-web/src/auth.ts`,
  `types.ts`).
- New pure helper `buildEndSessionUrl` (`packages/auth-web/src/logout.ts`) constructs the Keycloak
  `end_session_endpoint` URL with `id_token_hint` (fallback `client_id`) + `post_logout_redirect_uri`.
- Both logout actions (`apps/admin/app/layout.tsx`, `apps/applicant/components/PortalHeader.tsx`) now
  `signOut({ redirect: false })` then `redirect(buildEndSessionUrl(...))`, so Keycloak terminates the
  SSO session and returns to the app.
- Register the post-logout redirect on both Keycloak clients
  (`keycloak/import/talentos-realm.json` → `attributes."post.logout.redirect.uris"`): admin
  `http://localhost:3200/*`, applicant `http://localhost:3100/*`. The running realm was live-patched via
  `kcadm.sh` (non-destructive).

## Security

- Logout terminates the Keycloak SSO session for all roles/users, closing the "logout doesn't log out"
  hole.
- Post-logout redirect is restricted to each client's registered origin (verified: valid URI → 302,
  `evil.example.com` → 400).
- Accepted trade-off (D-050): the `id_token` is exposed via the user's own `/api/auth/session` — standard
  for RP-initiated logout; not a cross-user leak.
