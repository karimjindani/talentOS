# v0.18.3 Ops Regression Scenario Visibility

Code version: `v0.18.3`

## Summary

Enhance the local Ops Console so regression runs show individual scenario pass/fail/skipped results,
not only aggregate area cards. The regression runner already emits scenario-level JSON, so this slice
preserves and renders that payload in Ops.

## Key Changes

- Add `RegressionScenarioResult` to the shared Ops job contract.
- Parse `REGRESSION_RESULT_JSON.results` in `apps/ops/src/jobs.ts`.
- Store scenario results on the job step and top-level job.
- Render scenarios grouped by area in the Ops Console, with status pills, duration, detail and error
  text.
- Keep the existing area summary fallback when detailed scenario results are unavailable.

## Test Scenarios

| Scenario | Expected result | Automation |
| --- | --- | --- |
| Full regression emits detailed scenario results. | Ops job JSON includes every scenario with area, name, status and duration. | `apps/ops/src/jobs.test.ts` |
| Failed/skipped scenarios retain message text. | Ops can show the underlying failure/skip reason without requiring raw-log search. | `apps/ops/src/jobs.test.ts` |
| Existing summary-only payloads remain supported. | Older/partial regression output still renders summary cards. | Existing Ops job parser tests |
| Ops UI renders scenario-level cards. | Area cards include individual scenario rows and status pills. | Manual Ops validation |

## Out of Scope

- CI report publishing.
- Changing regression scenario definitions.
- Database schema or migration changes.
