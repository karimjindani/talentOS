# v0.11.3 — Keycloak Realm-Import Fix — Test Results

Code version: `v0.11.3` · Date: `2026-07-03`

Fixes a crash-looping Keycloak caused by an invalid `serviceAccountClientRoles` field on the
`talentos-provisioner` client in `keycloak/import/talentos-realm.json` (D-057). This is a Keycloak
realm-configuration fix; it is validated by a deployment / end-to-end test rather than the Vitest suite.

## Defect reproduced (before fix)

`docker logs talentos-keycloak` on startup:

```
ERROR: Failed to run import
ERROR: Unrecognized field "serviceAccountClientRoles" (class ...ClientRepresentation),
       not marked as ignorable ...
  ... RealmRepresentation["clients"]->ArrayList[4]->ClientRepresentation["serviceAccountClientRoles"]
```

Keycloak exits non-zero and restarts in a loop; `GET /realms/talentos/.well-known/openid-configuration`
returns connection-refused (`000`). Because the parse fails before the realm-exists check, the crash
occurs on **every** startup, fresh or existing.

## Fresh-import verification (after fix) — destructive, clean volume

Wiped the Keycloak database and re-imported from scratch:

```
docker compose stop keycloak keycloak-postgres
docker compose rm -f keycloak-postgres
docker volume rm talentos_keycloak-postgres-data
docker compose up -d keycloak-postgres keycloak     # start-dev --import-realm re-imports
```

| Check | Result |
|---|---|
| Keycloak boots | Passed — `Keycloak 26.0.8 ... started in 28.7s. Listening on http://0.0.0.0:8080` |
| Realm import error in logs | **None** (no `Failed to run import` / `Unrecognized field`) |
| OIDC discovery `GET /.well-known/openid-configuration` | 200 |
| Provisioner `client_credentials` token | Acquired |
| Admin API `GET /admin/realms/talentos/users` with that token | 200 (proves `manage-users`/`query-users` mapped) |
| `service-account-talentos-provisioner` user present | Yes |
