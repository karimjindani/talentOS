## Manual UI Verification (Keycloak browser flow)

The OIDC browser login cannot be scripted; verify manually against the running stack:

1. Sign in at `http://localhost:3100/login` as `applicant@demo.talentos.local` / `ChangeMe123!` →
   `/apply` → submit → `/application` shows status **SUBMITTED** and the motivation answer.
2. Sign in at `http://localhost:3200` as `hr@demo.talentos.local` / `ChangeMe123!` → `/applications`
   lists the application → open detail → **ACCEPTED** → list/detail reflect the new status.
3. Reload `/application` as the applicant → status now **ACCEPTED** with reviewer notes.
4. Negative: `techlead@demo.talentos.local` can enter the admin portal but the review action is denied
   (lacks `reviewApplications`); terminal applications expose no further actions.
