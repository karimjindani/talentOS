## Environment note (not a code defect)

During verification, browser logins failed machine-wide because the Windows hosts-file entry
`host.docker.internal → 192.168.100.119` was stale after a DHCP change (current host IP
`192.168.100.121`). Keycloak issuer URLs use `host.docker.internal` (`.env`), so the browser could
not reach the IdP. Restarting Docker Desktop rewrites the hosts entry. The test harness worked
around it with `--host-resolver-rules=MAP host.docker.internal 127.0.0.1`.
