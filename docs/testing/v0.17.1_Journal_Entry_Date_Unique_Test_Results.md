## Notes

- The `v0.18.0` test-results doc originally recorded `db:migrate` as blocked by "duplicate local
  Engineering Journal entries." Re-investigation found the `engineering_journal_entries` table did not
  exist in the local database used for this verification pass at all — none of the three migrations had
  been applied yet. Applying them together produced no conflict; the earlier report reflected a
  different, since-cleared local database state, not a defect in the `v0.17.1` migration or its
  `date_trunc('day', ...)` backfill step.
- The unique constraint is a database-layer backstop for a rule `packages/db/src/journal.ts` already
  enforced in application code; no behavior change is expected for callers going through the existing
  helpers.
