# AI Mentor Integration Test Cases — v0.15.0

**Code version:** `v0.15.0`  
**Branch:** `feat/applicant-ai-mentor-skeleton`  
**Date:** 2026-07-09  

---

## Overview

Integration tests verify the complete end-to-end flow through multiple system components. These tests require Docker containers running (Keycloak, PostgreSQL, Applicant Portal) and valid LLM API configuration.

**Flow:** User → API → Auth Guard → Context Builder → Knowledge Base → RBSE → LLM → Response → Persistence

---

## IT-01: Full LLM Integration Flow

| Field | Value |
| --- | --- |
| **ID** | IT-01 |
| **Name** | Authenticated user sends prompt and receives LLM response |
| **Preconditions** | Applicant logged in; LLM API key configured; Docker containers running |
| **Steps** | 1. POST `/api/ai/mentor` with `{ prompt: "Explain SDLC phases" }`<br>2. Verify auth guard passes<br>3. Verify context builder runs<br>4. Verify knowledge retrieval returns SDLC snippet<br>5. Verify RBSE classifies as `allow_llm` or `direct_answer`<br>6. Verify LLM is called (or direct answer returned)<br>7. Verify response has `status: "ok"` or `"stubbed"`<br>8. Verify response message is non-empty<br>9. Verify exchange persisted to DB |
| **Expected Result** | Status 200; response contains structured SDLC explanation; conversation persisted |
