## Overview

Edge case tests verify the AI Mentor handles boundary conditions, invalid inputs, and unexpected scenarios gracefully without crashing or exposing sensitive data.

---

## EC-01: Empty Prompt

| Field | Value |
| --- | --- |
| **ID** | EC-01 |
| **Scenario** | User submits an empty prompt |
| **Input** | `{ prompt: "" }` |
| **Expected Behavior** | API returns 400 with `{ error: "Prompt is required" }`; UI does not send request |
| **Risk** | Low — handled by both frontend trim check and backend validation |

## EC-05: Invalid API Key

| Field | Value |
| --- | --- |
| **ID** | EC-05 |
| **Scenario** | `GLM_Z_API_KEY` is set but invalid/expired |
| **Input** | Valid prompt with `GLM_Z_API_KEY=invalid-key` |
| **Expected Behavior** | LLM returns 401/403; `callGLM` throws `LLM_AUTH_ERROR`; no retry; fallback to stub response with degradation note |

| --- | --- |
| **ID** | EC-14 |
| **Scenario** | User from Tenant A tries to access Tenant B's conversation data |
