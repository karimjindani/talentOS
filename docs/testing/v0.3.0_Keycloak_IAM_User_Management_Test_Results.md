## RBAC Gating (automated)

| Request | Result |
| --- | --- |
| Applicant `/`, `/apply`, `/login` | 200 (public) |
| Applicant `/application` (gated) | 307 → `/login?callbackUrl=/application` |
| Admin `/` | 307 → `/api/auth/signin?callbackUrl=/` |
| Admin `/applications` | 307 → `/api/auth/signin?callbackUrl=/applications` |
| Admin `/forbidden` | 200 (open) |
| Applicant container `/applications` (admin route) | 404 (isolation intact) |

Admin gates every non-auth route to Keycloak sign-in; role differentiation (APPLICANT → `/forbidden`;
ORG_ADMIN/HR/TECH_LEAD/SUPER_ADMIN → admin routes) is enforced by `canEnterAdminPortal` and covered by
the unit tests.
