| --- | --- |

the unit tests.

## Manual Validation (browser OIDC) — step by step

### Test accounts (seeded in the realm + DB)

| Role | Email (username) | Password | Notes |
| --- | --- | --- | --- |
| Super Admin (platform) | `superadmin@talentos.local` | `ChangeMeSuper#1` | temporary → first login forces password change + TOTP |
| Org Admin | `orgadmin@demo.talentos.local` | `ChangeMe123!` | admin portal allowed |
| HR | `hr@demo.talentos.local` | `ChangeMe123!` | admin portal allowed |
| Tech Lead | `techlead@demo.talentos.local` | `ChangeMe123!` | admin portal allowed |
| Applicant | `applicant@demo.talentos.local` | `ChangeMe123!` | admin portal denied (`/forbidden`) |

Keycloak admin console: `http://localhost:8080` → `admin` / `admin`.

### A. Applicant login (`http://localhost:3100`)

1. Open `http://localhost:3100/` — public landing, no login required.
2. Visit `http://localhost:3100/application` (or click **Login**) → auto-redirect to `/login`.
3. Click **Sign in with Keycloak** → redirected to Keycloak (`host.docker.internal:8080`).
4. Enter `applicant@demo.talentos.local` / `ChangeMe123!`.
5. Redirected back to `/application` as a signed-in user; header shows the email + **Logout**.
6. **Logout** returns to the public landing.

### B. Admin login + RBAC (`http://localhost:3200`)

1. Open `http://localhost:3200/` → auto-redirect to Keycloak sign-in.
2. Denied: sign in as `applicant@demo.talentos.local` → lands on `/forbidden`.
3. Allowed: sign in as `orgadmin@demo.talentos.local` (or `hr@`, `techlead@`) → admin routes load; the
   sidebar shows the email, role and **Logout**.

### C. Super Admin first login (password policy + 2FA)

1. Open `http://localhost:3200/` → Keycloak sign-in.
2. Enter `superadmin@talentos.local` / `ChangeMeSuper#1`.
3. Keycloak forces **Update Password** (must satisfy: ≥12 chars, upper + lower + digit + special, not
   equal to the username).
4. Keycloak forces **Configure OTP**: scan the QR with an authenticator app and enter the 6-digit code.
5. Lands in the admin portal with `SUPER_ADMIN` (all routes).

### Switching users

Keycloak holds one SSO session per browser. To test a different role cleanly, use a separate
incognito window per user, or click **Logout**, or end the Keycloak session at
`http://localhost:8080/realms/talentos/account` → Sign out.
