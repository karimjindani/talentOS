## Defects Found During Testing

- The initial fix attempt kept the Host-derived `post_logout_redirect_uri`; live Keycloak rejected it
  with "Invalid redirect uri" from `demo.lvh.me` — confirming hostname wildcards
  (`http://*.lvh.me:3100/*`) never match. This produced the final canonical-host + `state` bounce
  design and revealed that tenant-subdomain logout had been broken in both portals since `v0.12.1`.
