## Manual UI Verification (Keycloak browser flow)

1. Sign in to the admin portal as `orgadmin@demo.talentos.local` / `ChangeMe123!` → `/programs` shows
   the seeded PUBLISHED and DRAFT programs.
2. Create a program (New program) → redirected to its edit page → **Publish** it.
3. As an applicant (`applicant@demo.talentos.local`), the published program appears in the `/apply`
   program list; **Archive** it in admin → it disappears from `/apply`.
4. Negative: `hr@demo.talentos.local` can view `/programs` but sees no New/Edit controls and the
   create/edit server actions redirect to `/forbidden`; invalid status transitions are rejected.
