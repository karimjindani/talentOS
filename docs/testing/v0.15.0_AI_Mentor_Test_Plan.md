## 1. Scope

This test plan covers the AI Mentor feature delivered across Phases 1–6 on branch `feat/applicant-ai-mentor-skeleton`. It validates the chat UI, API route, context builder, knowledge base retrieval, RBSE classifier, GLM/ZhipuAI LLM integration, conversation persistence, error handling, and fallback behavior.

### Objectives

1. Verify the AI Mentor chat UI renders correctly and provides a responsive, accessible experience.
2. Verify the API route (`/api/ai/mentor`) handles authentication, validation, context building, knowledge retrieval, LLM calls, and persistence.
3. Verify the RBSE classifier correctly blocks, directly answers, or allows LLM processing.
4. Verify the GLM/ZhipuAI integration produces complete, non-truncated responses within timeout.
5. Verify conversation persistence (localStorage + API history) works correctly.
6. Verify per-conversation loading state isolation (no shared loading across conversations).
7. Verify auto-scroll behavior on send and on new messages.
8. Verify fallback to stub responses when LLM is unavailable or API key is missing.
9. Verify no regressions in existing applicant dashboard, navigation, authentication, or tenant isolation.

## 3. Test Levels

### Unit Tests

Fast Vitest coverage for:
- RBSE classifier logic (`ai-rbse.ts`)
- Knowledge base retrieval (`knowledge-base.ts`)
- Context builder (`ai-context.ts`)
- AI interaction orchestrator (`ai.ts`)
- API route validation
- Conversation persistence utilities
- Nav item assertions (`ApplicantShell.test.ts`)

### Integration Tests

End-to-end flow:
- User sends prompt → API authenticates → Context built → Knowledge retrieved → RBSE classifies → LLM called → Response returned → Persisted

### Regression Tests

Scenario-based regression following `docs/Regression_Scenarios.md`:
- Existing applicant dashboard pages still functional
- Sidebar navigation intact (9 items including AI Mentor)
- Authentication and tenant isolation unaffected
- All existing regression areas pass

### Manual Tests

Step-by-step browser-based testing of:
- Chat UI interactions
- Conversation management (new, select, delete, clear)
- Loading states and UX indicators
- Error scenarios
- Edge cases
