## Known Limitations

- The applicant layout now hits the DB on every page load to resolve the tenant for CSS injection.
  This is acceptable for the current scale; a future iteration may cache at the edge.
- The existing `Tenant.logoUrl` column is now functionally unused (superseded by `logoFileId`).
  It is retained in the schema for this iteration and can be dropped in a future migration.
