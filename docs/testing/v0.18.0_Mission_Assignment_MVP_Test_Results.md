# v0.18.0 Mission Assignment MVP Test Results

## Scope

This test record covers the Mission Assignment MVP:

- Week 1 mission variants seeded from Markdown specs.
- `MissionAssignment` database model and uniqueness.
- Idempotent Week 1 assignment on accepted applications.
- Applicant mission visibility limited to assigned missions.
- Submission draft guard for unassigned missions.
- Dashboard progress based on assigned missions.
- Journal mission selection limited to assigned missions.

## Notes

The seeded Markdown specs are source material for local/demo data only. Seed imports the content into
existing `Mission` fields, so runtime pages and helpers read from the database.

Local migration note: `npm run db:migrate` could not apply the previous journal unique-date migration
because the local database contains duplicate `engineering_journal_entries` for the same tenant,
applicant and entry date. The duplicate local data must be resolved before the new
`mission_assignments` migration can be applied and the mission/dashboard regressions can pass against
that local database.
