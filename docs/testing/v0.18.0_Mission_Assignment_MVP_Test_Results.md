## Scope

- Week 1 mission variants seeded from Markdown specs.
- `MissionAssignment` database model and uniqueness.
- Idempotent Week 1 assignment on accepted applications.
- Applicant mission visibility limited to assigned missions.
- Submission draft guard for unassigned missions.
- Dashboard progress based on assigned missions.
- Journal mission selection limited to assigned missions.

## Notes

The seeded Markdown specs are source material for local/demo data only. Seed imports the content into
existing `Mission` fields, so runtime pages and helpers read from the database.

**Migration-blocker correction:** the original (2026-07-08) report attributed the `regression:missions`
/ `regression:dashboard` failures to duplicate local `engineering_journal_entries` rows blocking
`db:migrate`. Re-investigation on 2026-07-09 found the `engineering_journal_entries` table did not
exist at all in the local database used for re-verification — none of the `v0.17.0`/`v0.17.1`/`v0.18.0`
migrations had been applied yet, so there was nothing to conflict. Applying all three together
succeeded without incident, and both previously-failing regression areas now pass. The original failure
reflected a stale/dirty local database state on a different machine or session, not a defect in any of
the three migrations. See `D-074` and `D-075` in `docs/Decision_Log.md`.
