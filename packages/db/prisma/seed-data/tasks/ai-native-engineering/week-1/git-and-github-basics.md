# Git and GitHub Basics

## Objective

Use Git safely in a shared professional repository.

## Core Workflow

- Clone the repository and inspect the current branch with `git status`.
- Create a feature branch before changing files.
- Review changes with `git diff`.
- Stage intentional files with `git add`.
- Create a concise commit that explains the completed change.
- Fetch and integrate approved upstream work without overwriting teammates' changes.
- Push the feature branch and open a pull request for review.

## Collaboration Safety

- Do not push directly to main unless the repository policy explicitly allows it.
- Do not force-push a shared branch without approval.
- Do not discard changes you did not create.
- Resolve conflicts by understanding both sides of the change.

The goal is not only to memorize commands. It is to make every change reviewable, recoverable, and understandable to the team.
