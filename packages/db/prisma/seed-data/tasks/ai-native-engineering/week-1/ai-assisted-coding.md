# Introduction to AI-Assisted Coding

## Objective

Use AI as an engineering collaborator while keeping human judgment, testing, and accountability in the loop.

## Responsible Uses

- Clarify requirements and produce an implementation plan.
- Explore an unfamiliar codebase and identify existing patterns.
- Explain errors and suggest debugging experiments.
- Draft focused code and tests for review.
- Identify edge cases, security risks, and regression risks.
- Improve technical documentation.

## Required Engineering Judgment

- Verify generated code against the actual repository.
- Read and understand changes before accepting them.
- Run relevant tests, type checks, linting, and builds.
- Protect secrets, personal data, and tenant boundaries.
- Record how AI was used in the Engineering Journal.

AI can accelerate the work, but the applicant remains responsible for correctness, security, and evidence.
