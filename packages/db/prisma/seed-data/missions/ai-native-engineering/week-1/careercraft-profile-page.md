# Build a Recruiter-Ready Profile Page

## Objective

Convert a candidate's rough professional story into a deployed recruiter-ready profile page.

## Mission Brief

A junior engineer is applying for internships and needs a public profile page that helps recruiters quickly understand their skills, projects, learning goals, and contact options. The page is called CareerCraft.

Business context:
Recruiters often skim quickly. The client needs a clear, credible page that shows what the candidate can do, where to see proof, and how to contact them.

The page should help visitors understand:
- Who the candidate is
- What skills and projects they have
- What evidence supports their claims
- How to contact or shortlist them

You are expected to use AI tools, but you must document how AI helped you and what decisions you made independently.

SEM lifecycle:
1. Discover: write at least 5 clarification questions about candidate goals and recruiter needs.
2. Analyze: summarize the target users, trust signals, and content priorities.
3. Specify: create a short PRD with page goals, user stories, out-of-scope items, and acceptance criteria.
4. Design: create a page structure or wireframe optimized for recruiter scanning.
5. Build: build a responsive profile page with projects, skills, evidence links, and contact CTA.
6. Test: create and execute a manual test checklist.
7. Deploy: publish the site using Vercel, Netlify, GitHub Pages, or similar.
8. Present: record a 3-5 minute Loom demo.
9. Reflect: write an engineering journal entry.
10. Production Readiness Review: evaluate whether the page is ready to share with recruiters.

## Deliverables

- PRD
- User stories
- Acceptance criteria
- Wireframe or page structure
- GitHub repository URL
- Deployment URL
- README
- Manual test checklist
- Loom demo video
- Engineering journal entry
- AI usage disclosure

## Acceptance Criteria

- The profile page is publicly accessible.
- The page is responsive on desktop and mobile.
- The page includes project evidence, skill summary, and contact CTA.
- The GitHub repository contains a clear README.
- The PRD and user stories are included in documentation.
- The deployment URL works without login.
- AI usage is documented transparently.
- Manual testing evidence is included.

## Evaluation Criteria

Bronze: working deployed profile page with README and basic demo.
Silver: includes PRD, user stories, acceptance criteria, and test checklist.
Gold: recruiter-focused information architecture, polished responsive UI, and clear evidence links.
Platinum: professional personal-brand experience with accessibility checks, credible proof, and thoughtful production readiness reflection.
