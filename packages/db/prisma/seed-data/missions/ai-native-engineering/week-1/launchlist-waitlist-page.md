# Build a Product Waitlist Page

## Objective

Turn a rough product launch idea into a deployed waitlist page that captures early interest.

## Mission Brief

A founder is testing demand for a new productivity product called LaunchList. They need a public waitlist page that explains the offer, communicates benefits, and gives potential users a clear way to register interest.

Business context:
The client wants to validate demand before building a full app. The page must be credible enough for a real launch experiment, even if the signup mechanism is a simple form link or static mock.

The page should help visitors understand:
- What the product promises
- Who should join the waitlist
- Why joining early is useful
- What happens after they express interest

You are expected to use AI tools, but you must document how AI helped you and what decisions you made independently.

SEM lifecycle:
1. Discover: write at least 5 clarification questions about the launch, audience, and waitlist goal.
2. Analyze: summarize the target users, value proposition, and conversion risks.
3. Specify: create a short PRD with page goals, user stories, out-of-scope items, and acceptance criteria.
4. Design: create a landing/waitlist page structure or wireframe.
5. Build: build a responsive waitlist page with a clear CTA and realistic trust signals.
6. Test: create and execute a manual test checklist.
7. Deploy: publish the site using Vercel, Netlify, GitHub Pages, or similar.
8. Present: record a 3-5 minute Loom demo.
9. Reflect: write an engineering journal entry.
10. Production Readiness Review: evaluate whether the page is ready for a real launch experiment.

## Deliverables

- PRD
- User stories
- Acceptance criteria
- Wireframe or page structure
- GitHub repository URL
- Deployment URL
- README
- Manual test checklist
- Loom demo video
- Engineering journal entry
- AI usage disclosure

## Acceptance Criteria

- The waitlist page is publicly accessible.
- The page is responsive on desktop and mobile.
- The page has a clear value proposition, audience statement, and waitlist CTA.
- The GitHub repository contains a clear README.
- The PRD and user stories are included in documentation.
- The deployment URL works without login.
- AI usage is documented transparently.
- Manual testing evidence is included.

## Evaluation Criteria

Bronze: working deployed waitlist page with README and basic demo.
Silver: includes PRD, user stories, acceptance criteria, and test checklist.
Gold: compelling product positioning, responsive polished UI, and clear conversion-focused design.
Platinum: professional launch experiment with excellent copy, accessibility checks, clear evidence, and thoughtful production readiness reflection.
